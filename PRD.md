# 功能需求文档

## 让AI学会兄弟的人格

## MVP

### 前提

- 用户已经获取了他和好兄弟的聊天记录
- 用户已经配置了LLM

### 流程

1. 用户上传获取到的微信聊天记录。

2. 提交元prompt+解析后的聊天记录str，AI获取兄弟的人格，系统保存。

3. 用户发起和“兄弟”的会话，“兄弟”根据系统保存的人格，与用户进行对话。

## 高级功能

### 前端

除了配置API，其他所有操作都可以在web前端完成。

因此，前端设计到的具体功能为

- 用户SOUL创建
    1. 在index页，用户点击”创建SOUL“按钮。
    2. 用户上传获取到的微信聊天记录，系统启动解析。
    3. 系统解析完成后，把”发起蒸馏“的按钮交给用户。用户按按钮-发起推理请求。用户阻塞等待。
    4. 用户审核请求返回的人格，确认无误后，点击”保存“按钮；如果用户不喜欢这个人格，点击”重新解析“按钮，重新上传聊天记录。
- SOUL管理
    1. 在index页，用户点击”管理SOUL“按钮。进入SOUL管理页面，显示所有已创建的SOUL。
    2. 用户可以在管理页面选择删除SOUL。
    3. 用户可以在管理页面选择导出指定SOUL的markdown文件。
- 用户对话
    1. 在index页，用户点击”和bro对话“按钮。进入session选择列表。
    2. 用户选择session，系统加载该session的对话历史，渲染一个chat box。
    3. 用户可以在chat box中输入消息。按“发送”按钮后，系统会根据用户输入的消息，调用LLM生成回复。
    4. 系统会将回复渲染到chat box中，同时将用户输入的消息和回复保存到session的对话历史中。

### 非功能性需求

#### 聊天记录解析

- 解析用户传入的，可能为各种形式的聊天记录，变为格式化的str。

#### 存储

- BRO的souls的存储。当前放在/souls目录下。文件格式为markdown。

- 用户对话的存储。当前放在/sessions目录下。文件格式为json。
    json的格式
    - bro_name
    - chat_history
        - user_msg
        - bro_msg
        - user_msg
        - bro_msg
        - ... ...
